生成式深度学习的第一章：生成式建模

什么是生成式建模？

用概率模型来描述一个数据集的产生，这叫生成式建模。从中采样，我们就可以生成类似的新的数据。

可以理解成，一个数据集是符合一个未知的概率分布的，而生成式建模就需要模仿这种概率分布，以便之后采样，生成类似数据。

与分类模型区别

我们之前常见的比如图像分类都属于分类模型（Discriminative Modeling)，与生成模型区别开。

区别：分类模型属于监督学习，重点在“进行分类应用”；生成模型大多属于无监督学习，重点在于“生成新数据”。

生成模型会更重要的三个原因：

①探究数据如何产生
②驱使未来科技进步（如车自行模拟虚拟训练环境）
③更符合人类思想

朴素贝叶斯模型

朴素贝叶斯的大前提：各个属性相互独立。
我的理解：对于样本拥有很多属性的数据集。对每个属性的不同取值，进行计算得到一系列概率。最后，如果要计算一类样本（即所有属性的某种组合）的概率，则进行简单相乘即可。
对于书中讲到的例子（共五种属性，每种属性的取值类别数分别为7 6 3 4 8，则共需要系数个数=7+6+3+4+8-5=23，因为每种属性的概率总和为1，所以每种属性其实只需要取值个数-1）。
然后计算针对每种属性，不同取值出现的概率，最后对于一种属性取值组合实例，通过概率相乘来得到该实例的概率。即便该实例在数据集中未出现过，计算出的概率也可能不为0.

打破属性之间的相互独立性——引入生成式建模

将视线聚焦在“像素”层面，对于朴素贝叶斯是无法知道两个相邻元素之间的相似性的。
挑战：属性之间的高度依赖；如何找到令人满意的生成结果。
我们需要自行找出数据中的关联结构，而非人为提前假设。

表示学习：

Describe each observation in the training set using some
low-dimensional latent space and then learn a mapping function that
can take a point in the latent space and map it to a point in the
original domain.

表示学习其实是一种mapping，将低维隐层中生成一个点，映射到通用领域的具体结果中。
表示学习的好处在于可以学习到哪个特征最具代表性，从而对一个实例进行较精确的描述。
我的理解：不是仅停留在像素层面，而是建立隐层（可以更低维）的特征来表示图像，然后再对这些特征进行组合，从而生成新数据。
