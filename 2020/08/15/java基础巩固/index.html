类基础知识

main函数

  1. PSVM：public static void main(String[] args)
  2. 严格意义上说，main函数不属于这个类的所拥有的的函数，是“寄居”在一个class中。
  3. main函数也无法被其他方法/类所调用。
  4. main函数的形参可以传入.
  5. 在一个java文件中，可以有多个class定义，但只能有一个public class

自定义函数

  1. 不能以返回类型来区分同名函数。若只有返回值不同，将报错。
  2. 重载：函数名相同，形参列表一定不同。

对象

  1. 对象赋值是Reference（指针）赋值；基本数据类型是值传递。
  2. 关于函数的传值和传地址。如果是对象的话，肯定就是传地址。 输出结果：n=1m=2
    n=2m=1

构造函数

  1. 构造函数和类名是一样的。其次，构造函数是没有返回值的，也不能写void，否则只是普通函数。
  2. Java有构造函数，但没有析构函数。因为JVM有垃圾回收机制，不需要程序员操心析构函数。
  3. 一个类可以有多个构造函数，只要形参列表不相同即可。

信息隐藏原则

  1. 类的成员属性是private；类的方法是public，通过方法来修改成员属性的值。（打比方，朋友只能通过公开接口来修改和访问东西。）
  2. 可以通过IDE自动generate getter and setter。

继承

  1. 继承后，如果在构造函数第一句没有super()，则编译器会帮你加父类的无参构造函数super()。而自己写的话可以调用父类的有参构造函数。
  2. 父类的private是不给继承的。
  3. 单继承。

抽象类

  1. abstract。函数不给方法体，那么abstract函数所在的类也声明为abstract。当然，抽象类中是可以有具体方法的。
  2. 抽象类也是类，一个类继承于抽象类，就不能继承于其他类。
  3. 子类可以继承于抽象类，但是一定要实现父类们所有abstract的方法，若不能完全实现，则子类也是抽象类。

接口

  1. 如果类的所有方法都没有实现，那么这个类就是接口interface。（抽象类可以有具体方法，而接口不能）
  2. 类只可以extends一个类，但是可以implements多个接口。
  3. 实现接口也是要把所有方法都实现，否则就是抽象类。
  4. extends必须写在implements前面。

多态

类转型

  1. 类型可以相互转换，但是只限于有继承关系的类。
  2. 子类可以转换为父类，而父类不可转换为子类。因为子类可以从大变小向上转型（丢弃一些多余的）。
  1. 父类转为子类只有一种情况例外，就是这个父类本身就是从子类转化而来的。

多态的作用

  1. 以统一的接口来操纵某一类中不同的对象的动态行为。
    一定要注意，就算从子类转型为父类，但是指向的内存空间是不变的。所以，再次调用方法，仍是原来的子类类型中实现的具体方法。
  2. 对象之间的解耦。（基于接口，利用转型和多态，不影响真正方法的调用，成功地将调用类和被调用类解耦(decoupling)。）
    多态的一个典型使用方法，就是在函数形参中是统一的父类类型，但传入的实参可以是其不同的子类类型！

匿名类

直接new接口，但要马上补全方法实现。
匿名类只能用一次。

static

static变量

  1. 静态类变量是依赖于类存在的，而不依赖于对象实例存在。所以在构造函数中，只能输出静态类变量，除非新建实例。
  2. 所有的对象实例，其类变量也只是指向同一个内存空间。

static方法

  1. 可以通过类名直接引用。其中必须都是静态类变量。
  2. 非静态方法可以调用静态方法，而静态方法不可调用非静态方法。

static块

  1. 只在类第一次被加载时调用
  2. 执行顺序：static块>匿名块>构造函数

实例化对象，输出：22222 111111 444444 33333

单例模式

限定某个类在整个程序运行过程中，只能保留一个实例对象在内存空间。（一个类只有一个对象）

·设计模式：在软件开发过程中，经过验证的，用于解决在特定环境下的、重复出现的、特定问题的解决方案。
23种设计模式：创建型、结构型、行为型

·单例模式：保证一个类有且只有一个对象

  1. 采用static来共享对象实例
  2. 采用private构造函数，防止外界new操作

用我的话来说，单例模式就是为了让一个类有且只有一个对象——构造函数是private的，只能在类内调用，所以我就在类中新建static对象实例，外界通过方法来获取这个对象（注意这个获取方法getInstance也是static的，这样才能直接通过类名来调用）。而且外界无法再new实例。
